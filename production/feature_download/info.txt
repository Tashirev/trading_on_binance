Модуль чтения признаков с API binance.com и сохранения в базе данных postrges.
Используется отдельный ключ (tash).
С осторожностью:
	- признаки пишутся по порядку с ключём ticker,
	- при перезапуске обновить ticker_count в main_features.py,
	- не допускать перерывов в сохранении признаков, иначе обучение модели будет не корректным.
